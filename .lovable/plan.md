## Цель

Сформировать полноценную главу 3 дипломной работы «Этап разработки» в формате **DOCX** на русском языке, описывающую процесс создания сайта «Авто Premium» по 7 этапам, с реальными скриншотами интерфейса и фрагментами кода из проекта.

## Что войдёт в документ

Структура главы (каждый этап оформлен по единому шаблону: название → функциональность → назначение → реализация → технологии → скриншот интерфейса → ключевой код → пояснение → краткий вывод):

1. **Создание основной структуры сайта** — главная, Header, Footer, блоки услуг/цен/отзывов/контактов, карта.
2. **Добавление ИИ-диагностики** — интеграция Lovable AI Gateway (Gemini), серверная функция, обработка запроса/ответа.
3. **Реализация онлайн-записи** — форма booking, выбор услуги/даты/времени, сохранение в БД, уведомление в Telegram.
4. **Кабинет пользователя** — регистрация/авторизация через Supabase Auth, профиль, просмотр заявок.
5. **Административная панель** — управление заявками, услугами, пользователями (роли через `user_roles`).
6. **Аналитика для бухгалтерии** — метрики, графики (Recharts), экспорт CSV, пресеты периодов.
7. **Kanban-доска заказов** — колонки по статусам, drag-and-drop, обновление статуса с уведомлением.

После каждого этапа — краткий вывод. В конце главы — общий вывод по разработке.

## План работ

1. **Сбор материала из кодовой базы.** Прочитать ключевые файлы: `src/components/Header.tsx`, `HeroSection.tsx`, `ServicesSection.tsx`, `PriceSection.tsx`, `ReviewsSection.tsx`, `ContactsSection.tsx`, `Footer.tsx`, `src/routes/booking.tsx`, `diagnostics.tsx`, `login.tsx`, `profile.tsx`, `admin.index.tsx`, `admin.kanban.tsx`, `admin.analytics.tsx`, `src/server/diagnose.ts`, `notify-telegram.ts`, `src/hooks/useAuth.tsx`, `useIsAdmin.tsx`. Выбрать по 1 ключевому фрагменту кода (15–40 строк) для каждого этапа.

2. **Сбор скриншотов.** Через браузерную автоматизацию открыть в превью и сделать скриншоты страниц:
   - `/` (главная — Hero, услуги, цены, отзывы, контакты)
   - `/diagnostics` (ИИ-диагностика)
   - `/booking` (онлайн-запись)
   - `/login` и `/profile` (авторизация и кабинет)
   - `/admin` (админ-панель)
   - `/admin/analytics` (аналитика)
   - `/admin/kanban` (канбан)
   
   Для админ-страниц предварительно войти под админ-аккаунтом (понадобится подтверждение пользователя на вход).

3. **Генерация DOCX** через `docx-js` со следующими параметрами:
   - А4, поля 2 см, шрифт Times New Roman 14 pt, межстрочный 1.5 — стандарт пояснительной записки.
   - Стили заголовков H1/H2/H3, нумерация разделов «3.1, 3.2 …».
   - Скриншоты вставлены как `ImageRun` (масштаб по ширине ~15 см).
   - Фрагменты кода — моноширинным шрифтом (Consolas 11 pt) на сером фоне.
   - Подписи: «Рисунок 3.X — …» под каждым изображением.

4. **QA артефакта.** Конвертировать DOCX в PDF/изображения и просмотреть каждую страницу: проверить вёрстку, переносы кода, видимость скриншотов, отсутствие пустых страниц. При нахождении проблем — править скрипт и пересобирать.

5. **Выдача файла.** Сохранить как `/mnt/documents/Глава_3_Этап_разработки.docx` и приложить через `<lov-artifact>`.
